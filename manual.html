General usage

To get started, you need to enter both your internal and external IP. To make things easier, you can hit both Refresh buttons, to retrieve them automatically.

Then you need to tell CWSRestart, what it should do, when the server is dead. Hit Select... and navigate to the Server.exe inside your CubeWorld folder. If you are using a custom launcher (for instance to change the number of players), select the launcher instead.

Now we need to select, how CWSRestart from wich networks CWSRestart should try to access the server. If you are hosting a server for everyone, tick all three boxes. If you are hosting a server for your network only, leave Internet unchecked.

To fire up CWSRestart, click Start watcher. In the textbox below, you can enter how often CWSRestart should check your server. The progress bar will show when the next check is due.

Optional settings

With the Additional processes Button, you can bring up a dialog, that allows you to select processes, that are running with your server. When the server is dead, CWSRestart will not only terminate the Server.exe, but also the additional processes as well. You can add/remove processes with a doubleclick.

The UPnP-Wizzard can help you to set up port forwading. Just click Refresh and you can open/close the required ports. You need a UPnP enabled router to do this.

With Validate configuration you can check, if all settings are set.

The checkboxes below the log can be used to filter messages.

Statistics & Access control

CWSRestart can also keep track on a few numbers. The statistics module is disabled by default. You can start it with the Enable statistics button in the bottom right corner.

The statistics module can store collected data in a CSV file. Hit the Save statistics to... button, to select where CWSRestart should store the data.

Enabling the statistics module will grant you access to the IP Filter. You can add/remove IPs by double clicking, or by using the textbox on the bottom right. Beneath the textbox, you can select the filter mode:

  • Whitelist: Every player, that is not Whitelist will be kicked
  • Blacklist: Every player, that is on the Blacklist will be kicked

To use Access control, you need to run CWSRestart as administrator.

Advanced settings

There are some settings, that are only editable via the ServerService.dll.config in the CWSRestart folder.

  • DoNotRedirectOutput indicates, if the Server.exe output should be routed through CWSRestart.
  • IPService is the service used to retrieve the global IP.
  • Loopback is the IP used for the loopback check
  • Port can be used, if you are hosting the server on a diffrent port.
  • Timeout is the time that the Server.exe has to exit, before it gets killed
  • StatisticsInterval is the delay between two statistics-updates (in ms)
  • SaveStatisticsEvery is the frequency, how often statistics will be saved. Also, this is the frequency for the Whitelist/Blacklist checks. Basically these checks will run ever X statistics-updates.
